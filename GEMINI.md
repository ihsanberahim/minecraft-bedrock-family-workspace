# Gemini Instructions

## Storage of Development Plans and Specifications
- **Plans**: All implementation plans must be stored in the root `plans/` directory instead of `docs/superpowers/plans/`.
- **Specs**: All specifications and design documents must be stored in the root `specs/` directory instead of `docs/superpowers/specs/`.

## Directory Purpose
- The root `docs/` directory is reserved solely to store long-term project context (e.g. system design architecture, reference manuals, onboarding context). Short-term or task-specific implementation plans and specs belong in their respective root directories.
